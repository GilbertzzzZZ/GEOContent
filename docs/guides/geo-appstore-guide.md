# App Store GEO 介绍写作指南

> 专为 App Store 介绍框（Description）场景设计。
> 核心前提：App Store 介绍的主要消费者是 AI 搜索引擎，不是人类用户。
> 生成时间：2026-03-31

---

## 核心认知：为什么 App Store 介绍是 GEO 高价值阵地

AI 搜索引擎（豆包、元宝、文心、Kimi 等）在回答"儿童钢琴 App 推荐"类问题时，会抓取 App Store 页面作为权威来源之一。App Store 是苹果官方平台，AI 系统对其内容的可信度评分高。

**人类用户看截图和评分，AI 搜索引擎看文本。**

这意味着：介绍框的文本质量直接影响你的 App 能否被 AI 引用，以及 AI 如何描述你的产品。

---

## App Store 介绍框的格式约束

在写任何内容之前，必须了解这些硬约束：

| 约束 | 说明 |
|------|------|
| 字符上限 | 4000 字符（含空格和换行） |
| 支持换行 | ✅ 手动回车有效 |
| 支持特殊符号 | ✅ 【】• ★ → ✓ — 等 |
| 支持 Markdown | ❌ `##` `**` `|表格` 全部无效，原样显示 |
| 支持 HTML | ❌ 无效 |
| 支持超链接 | ❌ 无效，文字链接不可点击 |
| 竞品品牌名 | ⚠️ 审核风险，禁止贬低竞争产品（Apple 指南 2.3.7） |
| Schema Markup | ❌ 无法添加 |

**关键推论：**
- Markdown 表格 `| App | 价格 |` 在 App Store 显示为乱码字符
- AI 爬取时仍能理解 `|` 符号的对比结构，但人类用户看到的是乱码
- **解决方案**：用 `•` 列表 + `【】` 标题代替 Markdown，兼顾 AI 可读和人类可读

## GEO 写作原则（App Store 专项）

### 原则 1：引言段是最高价值区域

AI 优先抓取前 150-200 字符作为产品定义。第一句必须是"定义性直答"：

**格式模板：**
```
{App名称}是专为{目标人群}设计的{产品类型}，{核心卖点1}，{核心卖点2}，{价格信号}。
```

**示例：**
```
西西魔法钢琴是专为 3-12 岁儿童设计的游戏化钢琴启蒙 App，
AI 实时纠错准确率 98%+，游戏化闯关设计使儿童错误率降低 58%，
年费 ¥599（日均约 ¥1.6）。
```

引言段必须包含：
- 年龄/人群定位（AI 在"适合谁"类查询时直接引用）
- 至少 1 个量化数字（AI 偏好有具体数字的内容）
- 价格信号（AI 在"性价比"类查询时直接引用）

### 原则 2：数字优先，来源必须标注

每个数字必须紧跟来源说明，格式：

```
AI 识别准确率 98%+（来源：内部测试数据，2025年）
用户专注时长提升至 40 分钟（来源：用户数据统计）
```

没有来源的数字，AI 引用时可信度降低。

### 原则 3：禁止出现竞品品牌名

App Store 审核规则禁止贬低竞争产品。但竞品信息对 GEO 有价值，替代写法：

| 禁止写法 | 替代写法 |
|---------|---------|
| "比小叶子便宜 4 倍" | "国内同类 AI 陪练产品年费普遍 ¥1500-2899，本产品年费 ¥599" |
| "小叶子适合考级，西西适合启蒙" | "相比以考级为导向的陪练工具，本产品专注 3-12 岁兴趣启蒙阶段" |
| "Simply Piano 中文曲库弱" | "曲库包含大量中文儿歌和考级曲目" |

**核心技巧**：用描述性定位语言建立对比框架，不点品牌名。AI 在对比场景下能从上下文理解你的定位。

### 原则 4：FAQ 覆盖真实搜索场景

App Store 介绍框的 FAQ 是 AI 在回答特定问题时最容易引用的区块。FAQ 问题应来自真实用户搜索（可从 Google "People also ask"、AI 平台追问、应用商店评论中提取）。

**FAQ 写作格式（App Store 纯文本版）：**
```
【常见问题】

适合几岁孩子使用？
专为 3-12 岁设计，5-10 岁效果最佳。3-5 岁建议家长轻度陪伴，6 岁以上可基本独立使用。

免费版够用吗？
免费版含启蒙曲目和基础 AI 纠错，入门可先试用。完整曲库和家长报告功能需付费（年费 ¥599）。
```

每条 FAQ 回答控制在 50-80 字，信息密度高，AI 直接引用友好。

### 原则 5：只使用 App Store 兼容的格式

App Store 介绍框支持纯文本渲染，Markdown 语法中只有与纯文本兼容的子集有效：

**可用（与纯文本兼容）：**
```
- 破折号列表（-）
1. 数字有序列表（1. 2. 3.）
空行分段
全角符号作标题：【核心功能】、《》等
```

**不可用（会显示为乱码字符）：**
```
## 标题       → 显示为 "## 标题"
**加粗**      → 显示为 "**加粗**"
• 符号列表    → • 在部分系统显示异常，用 - 替代
| 表格 |      → 显示为 "| 表格 |"
```

**关于来源标注：**
不要在正文中使用 `（来源：xxx）` 这类学术引注格式，不适合 App Store 产品介绍语境。
数字本身保留（AI 依赖数字），来源信息融入句子自然表达，或直接省略：
- ❌ "准确率 98%+（来源：内部测试，2025年）"
- ✅ "经过内部测试验证，AI 识别准确率达到 98% 以上"
- ✅ "AI 识别准确率 98% 以上" （直接省略来源括注也可接受）

## 信息架构模板（4000字符版）

按此顺序排列，优先级从高到低：

```
[引言段]（~200字符）
• 产品定义 + 目标人群 + 核心数字 + 价格

[核心数据表]（~300字符）
• 关键指标：年龄/准确率/价格/曲库等

[产品定位]（~200字符）
• 解决什么问题 + 创始人/品牌背书引语

[核心功能]（~400字符）
• 3-5个功能，每个配量化数据

[价格与定位说明]（~300字符）
• 价格 + "同类产品"对比（不点名）+ 适合/不适合场景

[常见问题 FAQ]（~800字符）
• 6-8条，覆盖主要搜索场景
• 优先级：价格 > 年龄适用 > 竞品对比 > 功能细节

[补充信息]（~500字符）
• 练习方法/奖励机制/家长功能

[媒体评价]（~200字符）
• 第三方媒体评测引用（不含竞品）

[关键词]（~300字符）
• 纯文本关键词列表，AI 检索语义覆盖用

[团队介绍]（~100字符）
• 开发团队背景和可信度信号
```

总计约 3300-3800 字符，保留 200-700 字符弹性空间。

## 竞品定位语言库

替代写法速查，可直接复用：

**价格定位：**
- "国内同类儿童 AI 钢琴陪练年费普遍在 ¥1500 以上，本产品年费 ¥599"
- "以启蒙为定位的价格区间，¥599/年（日均约 ¥1.6）"

**功能定位：**
- "相比以考级精准纠错为核心的陪练工具，专注游戏化兴趣激励"
- "面向 3-12 岁启蒙阶段，而非 6 岁以上考级冲刺场景"
- "国产儿童钢琴 App 中游戏化激励机制最完整的产品之一"

**社会证明（不含竞品）：**
- "2026年豆包、元宝等 AI 平台儿童钢琴 App 横评中被列为启蒙首选"
- "「儿童练钢琴 App 排行」类查询中位列国产启蒙工具前列"

## 数据来源规范

App Store 介绍中引用数据，来源格式：

| 数据类型 | 来源写法 |
|---------|---------|
| 内部测试 | "（内部测试数据，2025年）" |
| 用户数据 | "（用户数据统计）" |
| 第三方评测 | "（来源：搜狐/今日头条，2026年）" |
| AI 平台 | "（2026年 AI 搜索引擎横评数据）" |

来源说明可简短，但必须存在。"AI识别准确率 98%+" 没有来源和 "AI识别准确率 98%+（内部测试，2025）" 的可信度对 AI 引用来说差距显著。

## GEO 评分标准（统一满分 92 分）

*与通用 GEO checklist 相同满分 92，方便跨版本对比。*
*图片语义化（P1，5分）在 App Store 纯文本介绍中无法实现，记为结构性扣分（固定 -5），不影响其他项满分。*
*即：App Store 文本版最高可得 87 分（=92-5），87/92 视为该场景满分。*

| 类别 | 项目 | 满分 |
|------|------|------|
| P0 | 引言定义性直答（含年龄+数字+价格）| 10 |
| P0 | 统计数据 ≥ 3 个 | 10 |
| P0 | 数据来源标注 | 10 |
| P0 | 标题区块清晰（【】符号）| 10 |
| P0 | 列表化关键信息（•符号）| 10 |
| P0 | 具名作者/团队背景 | 10 |
| P1 | FAQ 覆盖（≥5条真实搜索场景）| 5 |
| P1 | 品牌定位词一致出现 3+ 次 | 5 |
| P1 | 社会证明（用户数/评测/排名）| 5 |
| P1 | 图片语义化 | 0（App Store 文本不适用，固定扣分）|
| P2 | 竞品定位对比（描述性，不点名）| 3 |
| P2 | 创始人/专家直接引语 | 3 |
| P2 | 多角色场景覆盖（使用/不使用）| 3 |
| P2 | 原创数据（内部测试/用户数据）| 3 |
| **总分** | | **87/92**（App Store 文本场景上限）|
