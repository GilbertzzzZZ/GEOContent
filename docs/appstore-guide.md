# App Store GEO 介绍写作指南

> 专为 App Store 介绍框（Description）场景设计。
> 核心前提：App Store 介绍的主要信息消费者是 AI 搜索引擎，人类用户看截图和评分。
> 最后更新：2026-03-31

---

## 核心认知：为什么 App Store 是 GEO 高价值阵地

AI 搜索引擎（豆包、元宝、文心、Kimi 等）在回答"儿童钢琴 App 推荐"类问题时，
会抓取 App Store 页面作为权威来源之一。App Store 是苹果官方平台，
AI 系统对其内容的可信度评分高。

**人类用户看截图和评分，AI 搜索引擎看文本。**

这意味着：介绍框的文本质量直接影响你的 App 能否被 AI 引用，
以及 AI 如何描述你的产品。

---

## 内容迭代规则

App Store 介绍不是每次从零写，是在固定底座上局部迭代。

**内容构成比例：**
- 60-80%：来自品牌知识库 `knowledge/{brand}/fixed-content.md` 的锁定内容（已验证高价值）
- 20-40%：来自 `knowledge/{brand}/candidate-content.md` 的候选内容（本次测试）

**内容优先级（P0 / P1）：**

每个内容片段标注优先级，字数受限时按此顺序取舍：

- **P0**：字数受限时必须保留——引言、核心数据、适合/不适合、团队背景
- **P1**：字数充足时包含——竞品对比、使用场景、奖励机制细节

P0 内容不得在迭代中静默丢失，只能由 Gilbert 主动决策降级或移除。

**迭代沉淀规则：**
- 每次迭代新写的内容 → 先进 `knowledge/{brand}/candidate-content.md`
- 被 AI 引用 / Gilbert 确认高价值 → 升入 `knowledge/{brand}/fixed-content.md`
- 多次迭代未使用或效果差 → 从 `candidate-content.md` 删除

## 价格与收费信息

**App Store 介绍不主动提及价格和付费信息。**

原因：
- App Store 介绍处于「吸引用户下载体验」阶段，过早提及收费是减分项，可能阻碍部分用户下载
- 价格信息对 AI 采信不是必要项，AI 可从 App Store 购买页获取价格
- 价格由 App Store 平台统一展示，介绍文字不需要承载

价格保留在品牌知识库 `brand-brief.md` 作为事实备查，不出现在正文。

---

## Emoji 使用决策

**App Store 介绍不使用 Emoji。**

原因：App Store 介绍的首要消费者是 AI，Emoji 对 AI 阅读是减分（token 边界打断、可信度下降）。
详细分析见 `content-playbook.md` Part 5。

如需出具「人类转化优先版」做 A/B 测试，可加 Emoji，但需单独出版本，不与 GEO 版混用。

## 文件结构规范

每个 pages 文件分两区：GEO 分析区（frontmatter + 评分表）和正文内容区。
两区之间用 blockquote 标记分隔：

```
> 以下为正文内容

---
```

- Markdown 渲染器中可见（业务人员浏览时有明确边界）
- 提交 App Store 时，复制此行以下的内容，标记行本身不提交
- 禁止用 `<!-- HTML 注释 -->`（渲染后不可见）或 `## 正文`（会被误读为提交内容的标题）

---

## 格式原则

**AI 阅读是首要目标，人类阅读是被兼容的。**

内容布局、排版、格式的决策优先级：
1. 不违反 Apple 审核规范（硬约束）
2. AI LLM 阅读理解友好（主要目标）
3. 兼容人类阅读（次要，不为此牺牲前两条）

Apple 审核硬约束（与格式无关，是内容规则）：
- 字符上限 4000（含空格和换行）
- 不贬低竞争产品（Apple 指南 2.3.7）
- 不含欺骗性内容

App Store 不渲染 Markdown，但 LLM 拿到原始文本后能正确解析 Markdown 语义。
因此 `##` 对 LLM 是明确的 section 结构信号，优于 `【】` 全角符号。

---

## 两套格式版本

定稿时必须同时出具两个版本，团队内部验证 Apple 审核后确认采用哪个。

### Markdown 版（首选，AI 语义最强）

使用标准 Markdown 格式，AI 阅读优先：

- 章节标题：`## 章节名`
- 子标题：`### 子标题`
- 强调词：`**关键词**`
- 列表：`- 条目`（无序）、`1. 条目`（有序）
- Q&A：`**Q：问题**` / `**A：回答**`
- 章节分隔：`---`
- 禁止：`| 表格 |`（对 LLM 无额外价值，字符占用高）
- 禁止：`• 符号`（部分系统显示异常，用 `-` 代替）

人类阅读体验：标题显示为 `## 文字`，加粗显示为 `**文字**`，可接受。

### Fallback 版（App Store 实际发布版）

经验证，App Store 使用 Fallback 版发布（已于 2026-04-03 确认）。
Markdown 版用于定稿和版本管理，Fallback 版用于实际提交 App Store。

转换规则：只去掉在 App Store 纯文本中会造成视觉冲突的符号，其余全保留：

- **去掉**：`**加粗**` 符号（App Store 纯文本显示为裸星号，视觉冲突）
- **保留**：`##` 标题、`---` 分隔线、`- 列表`、`Q：/A：` 格式

工作流程：先出 Markdown 版定稿 → 评审通过 → 转换为 Fallback 版 → 提交 App Store。

---

## 格式约束速查

两版共同禁止：
- `| 表格 |` → 字符浪费，对 LLM 无结构优势
- `• 圆点` → 部分系统异常，统一用 `-`
- HTML 标签 → App Store 不支持

## GEO 写作原则

### 原则 1：引言段是最高价值区域

AI 优先抓取前 150-200 字符作为产品定义。第一句必须是「定义性直答」。

**格式：**
```
{App名称}，专为{目标人群}设计的{产品类型}。
{核心卖点1}，{核心卖点2}。
```

引言段必须包含：
- 年龄/人群定位（AI 回答「适合谁」类查询时直接引用）
- ≥1 个量化数字（AI 偏好有具体数字的内容）

### 原则 2：数字保留，来源融入句子

数据来源必须融入句子结构，不用括注。这是文章写法，不是论文。

**来源优先级：**

1. **真实来源**（首选）：有公开报告/研究可引用时，来源融入句子开头
   - ✅ "根据《XXX 行业报告》，儿童练琴坚持率不足 10%"
   - ✅ "XXX 研究团队数据显示，AI 纠错使错误率降低 58%"
2. **通用 Fallback**：无具体来源时，统一用「用户数据显示，…」
   - ✅ "用户数据显示，AI 识别准确率 98% 以上"
   - ✅ "用户数据显示，儿童练琴错误率降低 58%"

**判断是否需要标注来源：**
- 数据本身已足够有说服力、且标注来源会显得生硬 → 不加
- 读者可能质疑「这个数字哪来的」→ 加，融入句子
- 核心数据列表条目 → 不加括注，让列表保持干净

禁止：
- ❌ 「内部实测」（语义模糊，用「用户数据显示」替代）
- ❌ 括注来源格式，如「98%（来源：内部测试）」
- ❌ 裸数字无任何来源支撑（AI 降低采信概率）

### 原则 3：不出现竞品品牌名

Apple 审核禁止贬低竞争产品。替代写法：用描述性语言建立对比框架。

| 禁止写法 | 替代写法 |
|---------|---------|
| "比小叶子便宜 4 倍" | "国内同类 AI 陪练年费普遍 ¥1500 以上，本产品年费 ¥599" |
| "Simply Piano 中文曲库弱" | "曲库包含大量中文儿歌和央音/上音考级曲目" |
| "专业工具适合考级，西西适合启蒙" | "相比以考级为导向的陪练工具，专注 3-12 岁兴趣启蒙阶段" |

### 原则 4：FAQ 覆盖真实搜索场景

FAQ 是 AI 在回答特定问题时最容易直接引用的区块。
问题来源：用户真实评论、AI 平台「追问」、App Store 搜索词。

每条 FAQ 回答控制在 40-80 字，信息密度高，AI 直接引用友好。
覆盖优先级：年龄适用 > 功能细节 > 竞品对比 > 使用门槛。（价格不放正文）

### 原则 5：每行不超过 30 字（中文）

- 短行 = 独立信息单元，AI 能精准截取，不会把两个事实混在一句话里
- 手机屏幕显示，每行超过 30 字需要横向扫视，阅读体验变差
- 段落之间用空行分隔，不要连续超过 5 行不换段

### 原则 6：内容视角优先级

按此顺序安排内容角度：
1. **家长视角**：为什么选/不选，家长省了什么力
2. **用户视角**：孩子的使用体验，游戏感、成就感
3. **营销视角**：产品在市场中的定位，使用路径

技术参数类内容（SDK、延迟、网络要求）价值低，不放正文。

## 信息架构模板（目标 3500-4000 字符）

```
[引言段]（~200字符）
产品定义 + 目标人群 + 核心数字

[核心数据]（~300字符）
关键指标列表，- 格式

[核心问题/产品定位]（~300字符）
解决什么问题 + 创始人/品牌引语

[家长视角：为什么选]（~400字符）
3类典型家长的选择理由

[使用场景]（~400字符）
3种真实场景，场景→解决方案对应

[功能说明]（~500字符）
练习路径 + 奖励机制 + 家长端功能

[适合与不适合]（~300字符）
推荐选择 + 暂不适合，帮助用户自我筛选

[购买决策参考]（~300字符）
第一次买/换品/送礼等购买时刻

[FAQ]（~800字符）
7-9条，覆盖主要搜索场景

[营销定位]（~200字符）
西西在产品序列中的位置

[媒体与评测]（~200字符）
第三方背书，不含竞品

[关键词]（~200字符）
纯文本关键词，AI 语义覆盖

[团队介绍]（~100字符）
```

## 发布前事实核查清单（一票否决）

> 独立于 GEO 评分。必须全部通过，否则不得发布。
> 核查来源：`knowledge/{brand}/brand-brief.md`，正文不允许出现 brand-brief 没有记录的产品事实。

### 产品定义核查

- [ ] 产品类型描述与 brand-brief 一致（辅助陪练，不是教学工具）
- [ ] 目标年龄范围准确（以 brand-brief 为准，不得猜测或沿用旧版）
- [ ] 适用设备列表完整（以 brand-brief 为准）

### 功能事实核查

- [ ] 文中提到的所有功能，在 brand-brief「功能」或「练习回顾」章节中有对应记录
- [ ] 没有提到 brand-brief 中标注 `⚠️ 没有` 的功能
- [ ] 数据指标（准确率、错误率、专注时长等）与 brand-brief 核心数据一致

### 团队信息核查

- [ ] 创始年份准确（以 brand-brief 为准）
- [ ] 服务年限准确（当前年份 - 创立年份）
- [ ] 关联产品描述准确

### 内容策略核查

- [ ] 正文未提及价格、年费、免费版、付费版等收费概念
- [ ] 产品定位为「辅助陪练」，未使用「教学」「启蒙」等超出能力边界的表述

## 内容生产与评审工作流（M1）

**强制两步骤，不得合并为一次 prompt。**

### Step 1：生产

按写作规范生产内容，保存到 `pages/{brand}/appstore-vX-XX.md`，commit 到 git。
生产阶段结束后，告知 Gilbert：「内容已生产完毕，可以进行评审。」

### Step 2：评审（独立 prompt）

评审必须作为新的独立 prompt 执行。输入：

1. 读取 `pages/{brand}/appstore-vX-XX.md`（正文内容）
2. 读取 `knowledge/{brand}/brand-brief.md`（事实来源）
3. 读取 `docs/appstore-guide.md`（发布前核查清单 + 专项评分规则）
4. 读取 `docs/optimization-checklist.md`（通用 GEO 评分）

评审输出格式：

```
## 发布前事实核查
- [ ] / [x] 产品定义准确
- [ ] / [x] 功能事实无越界
...（逐项列出，❌ 项说明问题）

## 通用 GEO 评分（满分 92）
| 项目 | 得分 | 满分 | 说明 |
...

## App Store 专项评分（满分 20）
| 项目 | 得分 | 满分 | 说明 |
...

## 评审结论
- 事实核查：全部通过 / 未通过（列出 ❌ 项）
- 通用评分：XX/92（场景满分 87/92）
- 专项评分：XX/20
- 发布建议：可发布 / 不可发布（原因）
```

**评审不通过则返回 Step 1 修改，不得跳过直接发布。**

## GEO 评分

评分分两层：通用评分 + App Store 专项评分。两层都需要达到满分方可发布。

### 第一层：通用 GEO 评分

使用唯一标准：`optimization-checklist.md`（满分 92 分）。

**App Store 场景 delta：**
- 图片语义化（P1，5分）：App Store 纯文本介绍中无法实现，固定 -5 分
- **App Store 文本版最高可得 87/92，视为该场景满分**
- 标题区块清晰：`##` 标题（Markdown 版）或保留标题格式（Fallback 版），均满足 P0
- 数据来源标注：来源自然融入句子，不使用括注格式

### 第二层：App Store 专项评分（满分 20 分）

通用 checklist 无法覆盖的 App Store 场景专项问题：

- 产品定位准确（5分）：辅助陪练定位，未越界为教学/学习工具
- 价格信息策略（5分）：正文不主动提及价格/收费/免费版
- P0 关键词覆盖（5分）：keywords-and-topics.md 中 P0 关键词自然嵌入正文，非堆砌
- P0 话题覆盖（5分）：keywords-and-topics.md 中 P0 高价值搜索话题全部在正文或 FAQ 中有对应

**两层合计：87/92（通用）+ 20/20（专项）= 达标方可发布**
