# GEO 优化 Checklist

> **唯一评分标准**，适用于所有内容类型。场景专项 guide 只说明与本标准的 delta，不重复定义评分。
> 逐项核对，每篇内容发布前过一遍。
> 评分规则：P0 项每项 10 分，P1 项每项 5 分，P2 项每项 3 分。满分 92 分。
>
> **App Store 场景 delta**：图片语义化（P1，5分）固定 -5，场景满分 87/92。详见 `appstore-guide.md`。

---

## P0：不做就别发（6 项，60 分）

- [ ] **引言直答**：第一段直接回答目标问题，用"X 是 Y"定义性句式开头，150 字以内
  - 验证：把第一段单独拿出来，能不能回答标题提出的问题？
- [ ] **统计数据 ≥ 3 个**：包含至少 3 个具体数字/百分比/量化结果
  - 验证：Ctrl+F 搜 "%"、"倍"、数字，数一下
- [ ] **来源标注**：每个数据和关键声明都标注了出处
  - 验证：删掉所有来源标注，内容是否变成了"自说自话"？如果是，说明标注到位
- [ ] **标题层级清晰**：使用 H2/H3 组织内容，每个 H2 下聚焦一个子话题
  - 验证：只看标题能不能理解全文结构？
- [ ] **列表化关键信息**：核心要点用子弹点或有序列表呈现，非散文式
  - 验证：有没有超过 200 字的纯散文段落？有就拆
- [ ] **具名作者**：标注作者真实姓名、职称、专业背景
  - 验证：作者信息在页面上是否可见（不只是 meta tag）？

**P0 得分：_____ / 60**

## P1：显著提升竞争力（4 项，20 分）

- [ ] **FAQ 覆盖**：包含 3-5 个 FAQ，用用户自然语言提问，每个回答 50-100 字
  - 验证：这些问题是用户真的会搜的吗？（检查 Google "People also ask"）
- [ ] **品牌定位词**：核心定位短语在文中自然出现 3+ 次
  - 验证：Ctrl+F 搜品牌定位词，出现次数 ≥ 3，且不显得堆砌
- [ ] **社会证明**：至少 1 个社会认同元素（用户数/评价/排名/客户 logo）
  - 验证：读者能否从文中判断"有多少人认可这个品牌/产品"？
- [ ] **图片语义化**：所有图片有描述性 Alt Text 和 Caption
  - 验证：关闭图片显示，Alt Text 能否传达图片信息？

**P1 得分：_____ / 20**

---

## P2：拉开差距（4 项，12 分）

- [ ] **对比表格**：至少 1 个数据对比表格（产品/方案/特征比较）
- [ ] **专家引语**：至少 1 条具名专家的直接引语
- [ ] **多角色覆盖**：内容兼顾学习者/决策者/比较者不同视角
- [ ] **原创数据**：包含品牌原创研究、独家发现或第一手数据

**P2 得分：_____ / 12**

---

## 技术检查（通过/不通过，不计分但一票否决）

- [ ] **robots.txt**：GPTBot、CCBot、Claude-Web、Google-Extended 未被屏蔽
- [ ] **SSR/预渲染**：页面核心内容在禁用 JS 后仍可见
- [ ] **Schema Markup**：已部署 FAQPage / HowTo / Article / Product（至少 1 个）
- [ ] **Bing 索引**：页面已被 Bing 收录（ChatGPT Search 的前提条件）
- [ ] **无登录墙**：核心内容对爬虫可见，无需登录
- [ ] **Canonical 正确**：canonical 标签指向正确 URL

## 快速修复优先级

当时间紧迫时，按这个顺序修：

1. 技术检查未通过项（10 分钟，但决定生死）
2. 引言段改为直答式（15 分钟，影响最大的单一改动）
3. 补统计数据和来源标注（30 分钟，可见度提升 30-40%）
4. 加 FAQ 区域（20 分钟，覆盖更多长尾查询）
