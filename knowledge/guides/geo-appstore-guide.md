# App Store GEO 介绍写作指南

> 专为 App Store 介绍框（Description）场景设计。
> 核心前提：App Store 介绍的主要信息消费者是 AI 搜索引擎，人类用户看截图和评分。
> 最后更新：2026-03-31

---

## 核心认知：为什么 App Store 是 GEO 高价值阵地

AI 搜索引擎（豆包、元宝、文心、Kimi 等）在回答"儿童钢琴 App 推荐"类问题时，
会抓取 App Store 页面作为权威来源之一。App Store 是苹果官方平台，
AI 系统对其内容的可信度评分高。

**人类用户看截图和评分，AI 搜索引擎看文本。**

这意味着：介绍框的文本质量直接影响你的 App 能否被 AI 引用，
以及 AI 如何描述你的产品。

---

## 格式硬约束

| 约束 | 说明 |
|------|------|
| 字符上限 | 4000 字符（含空格和换行） |
| 支持换行 | ✅ 手动回车有效 |
| 支持特殊符号 | ✅ 【】★ → ✓ — 等全角符号正常显示 |
| 支持 Markdown | ❌ `##` `**` `|表格` 全部原样显示，不渲染 |
| 支持 HTML | ❌ 无效 |
| 竞品品牌名 | ⚠️ 审核风险，禁止贬低竞争产品（Apple 指南 2.3.7） |

**可用格式（与纯文本兼容）：**
- `- 破折号列表`（推荐，App Store 兼容）
- `1. 有序数字列表`
- 空行分段
- 全角符号标题：`【核心功能】`

**不可用（显示为乱码字符）：**
- `## 标题` → 原样显示
- `**加粗**` → 原样显示
- `| 表格 |` → 原样显示
- `• 符号` → 部分系统显示异常，统一用 `-` 替代

## GEO 写作原则

### 原则 1：引言段是最高价值区域

AI 优先抓取前 150-200 字符作为产品定义。第一句必须是「定义性直答」。

**格式：**
```
{App名称}，专为{目标人群}设计的{产品类型}。
{核心卖点1}，{核心卖点2}。年费{价格}，日均约{折算}。
```

引言段必须包含：
- 年龄/人群定位（AI 回答「适合谁」类查询时直接引用）
- ≥1 个量化数字（AI 偏好有具体数字的内容）
- 价格信号（AI 回答「性价比」类查询时直接引用）

### 原则 2：数字保留，来源自然融入

每个关键数字尽量有来源支撑，但不使用学术引注格式：

- ❌ "准确率 98%+（来源：内部测试，2025年）"
- ✅ "经内部测试验证，AI 识别准确率达到 98% 以上"
- ✅ "AI 识别准确率 98% 以上"（数字本身即信号，无括注也可接受）

括注风格不适合 App Store 产品介绍语境，会破坏阅读流畅度。

### 原则 3：不出现竞品品牌名

Apple 审核禁止贬低竞争产品。替代写法：用描述性语言建立对比框架。

| 禁止写法 | 替代写法 |
|---------|---------|
| "比小叶子便宜 4 倍" | "国内同类 AI 陪练年费普遍 ¥1500 以上，本产品年费 ¥599" |
| "Simply Piano 中文曲库弱" | "曲库包含大量中文儿歌和央音/上音考级曲目" |
| "专业工具适合考级，西西适合启蒙" | "相比以考级为导向的陪练工具，专注 3-12 岁兴趣启蒙阶段" |

### 原则 4：FAQ 覆盖真实搜索场景

FAQ 是 AI 在回答特定问题时最容易直接引用的区块。
问题来源：用户真实评论、AI 平台「追问」、App Store 搜索词。

每条 FAQ 回答控制在 40-80 字，信息密度高，AI 直接引用友好。
覆盖优先级：价格 > 年龄适用 > 竞品对比 > 功能细节 > 使用门槛。

### 原则 5：每行不超过 30 字（中文）

- 短行 = 独立信息单元，AI 能精准截取，不会把两个事实混在一句话里
- 手机屏幕显示，每行超过 30 字需要横向扫视，阅读体验变差
- 段落之间用空行分隔，不要连续超过 5 行不换段

### 原则 6：内容视角优先级

按此顺序安排内容角度：
1. **家长视角**：为什么选/不选，家长省了什么力
2. **用户视角**：孩子的使用体验，游戏感、成就感
3. **营销视角**：产品在市场中的定位，使用路径

技术参数类内容（SDK、延迟、网络要求）价值低，不放正文。

## 信息架构模板（目标 3500-4000 字符）

```
[引言段]（~200字符）
产品定义 + 目标人群 + 核心数字 + 价格

[核心数据]（~300字符）
关键指标列表，- 格式

[核心问题/产品定位]（~300字符）
解决什么问题 + 创始人/品牌引语

[家长视角：为什么选]（~400字符）
3类典型家长的选择理由

[使用场景]（~400字符）
3种真实场景，场景→解决方案对应

[功能说明]（~500字符）
练习路径 + 奖励机制 + 家长端功能

[适合与不适合]（~300字符）
推荐选择 + 暂不适合，帮助用户自我筛选

[购买决策参考]（~300字符）
第一次买/换品/送礼等购买时刻

[FAQ]（~800字符）
7-9条，覆盖主要搜索场景

[营销定位]（~200字符）
西西在产品序列中的位置

[媒体与评测]（~200字符）
第三方背书，不含竞品

[关键词]（~200字符）
纯文本关键词，AI 语义覆盖

[团队介绍]（~100字符）
```

## 竞品定位语言库

**价格定位：**
- "国内同类儿童 AI 钢琴陪练年费普遍在 ¥1500 以上，本产品年费 ¥599"
- "年费 ¥599（日均约 ¥1.6），¥599 价位无直接平替"

**功能定位：**
- "相比以考级精准纠错为核心的陪练工具，专注游戏化兴趣激励"
- "面向 3-12 岁启蒙阶段，而非考级冲刺场景"
- "游戏化激励机制最完整的国产儿童钢琴 App 之一"

**社会证明（不含竞品名）：**
- "2026年豆包、元宝等 AI 平台儿童钢琴 App 横评中被列为启蒙首选"
- "「儿童练钢琴 App 排行」类查询中位列国产启蒙工具前列"

## GEO 评分标准（统一满分 92 分）

图片语义化（P1，5分）在 App Store 纯文本介绍中无法实现，
记为结构性扣分（固定 -5），不影响其他项满分。
**App Store 文本版最高可得 87/92，视为该场景满分。**

| 类别 | 项目 | 满分 | 备注 |
|------|------|------|------|
| P0 | 引言定义性直答（含年龄+数字+价格）| 10 | |
| P0 | 统计数据 ≥ 3 个 | 10 | |
| P0 | 数据来源标注 | 10 | 来源自然融入句子 |
| P0 | 标题区块清晰（【】符号）| 10 | |
| P0 | 列表化关键信息（- 列表）| 10 | 用 - 不用 • |
| P0 | 具名作者/团队背景 | 10 | |
| P1 | FAQ 覆盖（≥5条真实搜索场景）| 5 | |
| P1 | 品牌定位词一致出现 3+ 次 | 5 | |
| P1 | 社会证明（用户数/评测/排名）| 5 | |
| P1 | 图片语义化 | 0 | App Store 文本不适用，固定 -5 |
| P2 | 竞品定位对比（描述性，不点名）| 3 | |
| P2 | 创始人/专家直接引语 | 3 | |
| P2 | 多角色场景覆盖（使用/不使用）| 3 | |
| P2 | 原创数据（内部测试/用户数据）| 3 | |
| **总分** | | **87/92** | App Store 文本场景上限 |
